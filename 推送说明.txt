如何使用 GitHub Actions 构建：
==============================

方法 1：推送代码自动构建
-----------------------
每次推送到 main 分支，GitHub Actions 会自动开始构建

方法 2：创建标签触发发布
-----------------------
1. 创建标签：
   git tag v1.0.0
   git push origin v1.0.0

2. GitHub Actions 会自动：
   - 构建所有平台的安装包
   - 创建 GitHub Release
   - 上传安装包到 Release

方法 3：手动触发构建
-------------------
1. 访问：https://github.com/ggjjc786-boop/1111111111111111111111111111111111/actions
2. 选择 "Build and Release" 工作流
3. 点击 "Run workflow"
4. 选择分支并运行

构建产物：
=========

Windows:
- .msi 安装包
- .exe 安装包

macOS:
- .dmg 安装包（Intel）
- .dmg 安装包（Apple Silicon）

Linux:
- .AppImage 可执行文件
- .deb 安装包

注意事项：
=========

1. 首次构建可能需要较长时间（10-20分钟）
2. 后续构建会使用缓存，速度会更快
3. 如果需要签名，需要配置 TAURI_PRIVATE_KEY 和 TAURI_KEY_PASSWORD secrets
4. 构建产物会保留 7 天
